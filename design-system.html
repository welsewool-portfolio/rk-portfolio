Design
Systems

I have created multiple design systems for products of varying scales, either starting from scratch or with minimal existing resources.

How I create Design Systems

1
Conduct Visual
Audit & Research

The first step is taking inventory of the UI components, patterns, HTML/CSS code in use. Define main issues, pain points, hierarchy and usability gaps, inconsistency, and lack of standards. I also discover the competitive solutions and common approaches.

2
Prioritise and set
up Principles

I prioritize entities and create the roadmap for design system execution. Additionally, at this stage, I set up the visual language and principles of the Design System. Building Design System, I follow the Atomic Design Principles.

3
Create a UI/
Pattern library

I create a sufficient amount of the primitives, colors, typography, spacing and other rules to cover all the required components and major patterns which will be reused in different parts of the interface.

4
Code it

Next, I pass all the created design assets to the front end team and they make them live and reusable throughout the Interface. At this stage, I work in tight collaboration with developers.

5
Document it

At the final step I care about the Product Knowledge Transfer and develop design guidance, behaviours guide, elements usage rules, accessibility recommendations, code snippets, etc.

Design System
Research

I always prioritize research before diving into any design system project. My research process allows me to gain a deep understanding of the project's goals, target audience, and unique challenges, ensuring that the resulting design system is tailored to meet those needs.

Through research, I'm able to make informed decisions that lead to a successful and impactful design system.

Design System
Planning

Before starting design work I always plan the structure and full scope of the final delivery. This structure could be static or flexible enough to make changes during the process. I organize all the needed components and their states for clear structures, so I can see the nesting and hierarchy before I start actual development.

I also put the plan into a checklist to follow the progress and split tasks between team members.

Components
Hierarchy

Using the organized structures from the previous step I do all the needed components. I nest items into a logic hierarchy, so design system users could navigate fast and easily, having appropriate items within seconds.

I put as many parameters into account as designers usually use in a real product. That helps to reduce the customization work and turns it to work with logic and content only.

Color Contrast &
Accessibility

In my design systems, I prioritize color contrast and strive to maintain consistent rules for every color in the palette. This ensures that all UI elements are legible and visually accessible to users, regardless of their visual abilities.

Whenever possible or required, I also follow accessibility guidelines to ensure that all users can interact with the design system in an inclusive and equitable way.

Typography
System

I always plan ahead and create a flexible and comprehensive typography system that can cover all possible use cases in product interfaces.

This ensures that typography is consistent and legible across all devices and screen sizes, while also allowing for flexibility in design. By considering all possible use cases, I can create a typography system that is both functional and aesthetically pleasing.

Tokens &
Theming

In my design systems, I collaborate closely with front-end development teams to create custom token systems. These systems help to establish a shared language between designers and developers, enabling faster and more efficient design-to-development workflows.

By working together to define the tokens, we can ensure that the design system is both visually consistent and technically sound. This collaboration also allows for easier updates and maintenance of the design system over time.

Documentation

Documentation is a critical component of a design system, as it outlines the rules and guidelines that users must follow to ensure consistency and maintain the system's integrity. When preparing documentation for a design system, I provide clear and concise information on various aspects, including typography, color usage, layout, and interactions.

Proper documentation ensures that everyone who uses the design system can produce cohesive and high-quality work, regardless of their level of expertise.
