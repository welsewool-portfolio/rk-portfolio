Design Systems Workshop

Fleetmatics has already had a working product, but it was quite outdated and had a lot of usability problems. It wasn't user-friendly enough to have a real competitive advantage in front of other's solutions. Moreover, it wasn't ready for tablet and smartphone usage at all.

The goal
and the hypothesis

The main objective was to create a brand-new approach to interact with the data and to make it as simple as possible. The main idea behind was using more graphic solutions instead of numbers and texts only.

The new UI Model should have been flexible enough to provide users with the same experience for desktop, tablet, and smartphones.

The hypothesis was making one main dashboard with a lot of additional features on top of it so the experience could be improved.

How
I got there

After a detailed exploration of the current solution, I had mapped all the existing functionality and added the new features into the scheme. Firstly, I did the low-fidelity mapping to understand the overall picture and then went deeper with a high-fidelity map. Having all the functionality clarified I regrouped entities and organized them into the sitemap-like dashboard structure, so I had the future interface structure defined.

I did sketching sessions with a few stakeholders trying to find the best solution, created alternative layout models, conducted a few feedback sessions, and we chose one direction. I prepared high-fidelity wireframes for major flow. As a final activity, I executed the visual solution of the interface and prepared the interactive prototype and animation to demonstrate the product.

How I create Design Systems

1
Conduct Visual
Audit & Research

The first step is taking inventory of the UI components, patterns, HTML/CSS code in use. Define main issues, pain points, hierarchy and usability gaps, inconsistency, and lack of standards. I also discover the competitive solutions and common approaches.

2
Prioritise and set
up Principles

I prioritize entities and create the roadmap for design system execution. Additionally, at this stage, I set up the visual language and principles of the Design System. Building Design System, I follow the Atomic Design Principles.

3
Create a UI/
Pattern library

I create a sufficient amount of the primitives, colors, typography, spacing and other rules to cover all the required components and major patterns which will be reused in different parts of the interface.

4
Code it

Next, I pass all the created design assets to the front end team and they make them live and reusable throughout the Interface. At this stage, I work in tight collaboration with developers.

5
Document it

At the final step I care about the Product Knowledge Transfer and develop design guidance, behaviours guide, elements usage rules, accessibility recommendations, code snippets, etc.

And many other design activities depending on a specific task and project needs
