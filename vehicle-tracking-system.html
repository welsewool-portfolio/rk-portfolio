User Interface & Dashboard redesign for Verizon's Vehicle Tracking System

My Role & Responsibilities

My Role

User Experience Designer

I had to review & explore the current solution, define weak points, and offer an alternative UI Model to push the interface to the next stage of the Product roadmap. The company was transforming and joining the Verizon group, so I had to demonstrate growth opportunities.

My Responsibilities

  • Current solution exploration
  • Problem framing and pain point definition
  • Requirements gathering
  • Mapping functionality
  • User scenarios
  • New vision statement
  • Planning product structure & UI Model
  • Sketching
  • Wireframing
  • Prototyping
  • Animation

The Challenge

Fleetmatics has already had a working product, but it was quite outdated and had a lot of usability problems. It wasn't user-friendly enough to have a real competitive advantage in front of other's solutions. Moreover, it wasn't ready for tablet and smartphone usage at all.

The goal

The main objective was to create a brand-new approach to interact with the data and to make it as simple as possible. The main idea behind was using more graphic solutions instead of numbers and texts only.

The new UI Model should have been flexible enough to provide users with the same experience for desktop, tablet, and smartphones.

The hypothesis was making one main dashboard with a lot of additional features on top of it so the experience could be improved.

How I got there

After a detailed exploration of the current solution, I had mapped all the existing functionality and added the new features into the scheme. Firstly, I did the low-fidelity mapping to understand the overall picture and then went deeper with a high-fidelity map.

Having all the functionality clarified I regrouped entities and organized them into the sitemap-like dashboard structure, so I had the future interface structure defined.

I did sketching sessions with a few stakeholders trying to find the best solution, created alternative layout models, conducted a few feedback sessions, and we chose one direction. I prepared high-fidelity wireframes for major flow. As a final activity, I executed the visual solution of the interface and prepared the interactive prototype and animation to demonstrate the product.
